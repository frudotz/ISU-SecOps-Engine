# Core Architecture

Bu modül, web sitelerinin HTTP güvenlik headerlarını analiz ederek eksik veya hatalı yapılandırmaları tespit etmeyi amaçlamaktadır.

## Genel Akış

1. Kullanıcı CLI üzerinden hedef URL girer
2. Uygulama ilgili adrese HTTP isteği gönderir
3. Dönen response içerisindeki header'lar alınır
4. Tanımlı güvenlik header listesi ile karşılaştırma yapılır
5. Her header için:
   - Var/yok durumu kontrol edilir
   - Severity (önem derecesi) belirlenir
   - Eksikse olası risk açıklaması eklenir
6. Genel güvenlik skoru hesaplanır
7. Sonuçlar:
   - Terminalde Markdown formatında gösterilir
   - JSON olarak dosyaya kaydedilir

---

## Kullanılan Teknolojiler

- Rust
- reqwest (HTTP istekleri)
- tokio (async yapı)
- clap (CLI yönetimi)
- serde (JSON serialize)

---

## Veri Yapısı

### headerResult
Her header için detaylı bilgi içerir:

- name → header adı
- present → mevcut olup olmadığı
- severity → önem derecesi (HIGH / MEDIUM / LOW)
- risk → eksikse oluşabilecek risk

### report
Genel analiz çıktısı:

- url → hedef adres
- grade → harf notu
- score → 0–100 arası skor
- headers → header listesi

---

## Skorlama Mantığı

Sistem başlangıçta 100 puan ile başlar.

Eksik header'lara göre puan düşürülür:

- HIGH → -20
- MEDIUM → -10
- LOW → -5

Sonuç olarak elde edilen puan, harf notuna çevrilir:

| Skor | Not |
|------|-----|
| 90-100 | A |
| 75-89  | B |
| 50-74  | C |
| <50    | F |

## Güncelleme Notu (v0.3.0)

Bu sürüm ile birlikte sistem yalnızca header varlığını kontrol etmekle kalmayıp, aynı zamanda eksik header'lar için risk analizi yapabilen daha gelişmiş bir yapıya kavuşmuştur. Ayrıca severity tabanlı skor sistemi ile daha gerçekçi bir değerlendirme sunulmaktadır.
